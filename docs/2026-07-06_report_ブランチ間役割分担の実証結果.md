# レポート：main／featureブランチ間の「テスト役割分担」実証結果

| 項目 | 内容 |
|---|---|
| 報告元 | `claude/android-camera-pii-masking-io6ucl`（PrivacyCamera 機能開発ブランチ、Sonnet 5 セッション） |
| 宛先 | istech標準開発環境セッション（`main`、Fable 5 セッション） |
| 目的 | 「mainにテスト環境が完成したら、featureブランチに取り込まず、mainにテストを引き渡して結果だけ受け取る」役割分担が可能かの実証結果を共有し、標準開発環境の設計に反映してもらう |
| 日付 | 2026-07-06 |

---

## 1. 検証した問い

mainブランチに Robolectric + Roborazzi によるテスト基盤（unit/golden/screenshotテスト、istechブランド準拠のオフラインポリシー検査ステップなど）が整備されたことを受け、
「このテスト環境を、機能開発ブランチ（本ブランチ）にマージすることなく、`workflow_dispatch` 経由で本ブランチのコードに対して走らせ、結果だけを受け取る」という非対称な役割分担が技術的に可能かを実験した。

## 2. 実験内容と結果

- 本ブランチの `android.yml`（旧来の、Roborazziを含まない版）に対して `workflow_dispatch` で `ref: claude/android-camera-pii-masking-io6ucl` を指定して手動起動。
- 実行された run（id `28763267660`, head_sha `8119d012`）のジョブステップを確認したところ、実行されたのは**本ブランチ自身の `.github/workflows/android.yml`**（Checkout → 独自の静的チェック → JDK/SDKセットアップ → ビルド → アップロード）のみで、mainにのみ存在する `Run unit & golden tests` ステップやistechブランドのオフラインポリシー検査ステップは**一切実行されなかった**（成功はしたが、テスト基盤が適用されていない状態での成功）。

## 3. 結論

**GitHub Actionsの `workflow_dispatch` は、指定した `ref` がその時点でチェックアウト状態に持つワークフローYAMLをそのまま実行する。** デフォルトブランチ（main）側の定義を「差し込んで」実行することはない。

したがって、当初想定していた「mainがテストを実行し、featureブランチは結果だけ受け取る」という完全非対称な役割分担は、**現状の運用（featureブランチがmainのワークフロー更新を取り込んでいない）では成立しない**。テスト基盤はワークフローファイル・Gradle設定・テストコードとして各ブランチのworking treeに物理的に存在しない限り、そのブランチのCIには一切効かない。

## 4. 本ブランチからの提案（標準開発環境設計への反映依頼）

今後、新規アプリ開発のたびに新規セッション・新規ブランチを立てる運用を続ける前提で、以下の方針を標準開発環境側の設計に組み込むことを提案する。

1. **新規ブランチは常に最新のmainから分岐する。**
   mainが「テスト・CI・Gradle規約などの標準環境」を保持する一本のtrunkとして機能し、新規アプリ開発ブランチはその時点のmainの状態（＝標準環境一式）を継承した状態でスタートする。
2. **長期化するfeatureブランチは、定期的にmainをマージして標準環境の更新を取り込む。**
   逆方向（feature→main）の統合を待つのではなく、mainの更新を都度小さく同期する運用の方が、今回のような大規模な事後統合より低コストで済むと考えられる。
3. **（任意・効果は限定的）共通CIロジックの `workflow_call` 化。**
   各アプリのワークフローYAMLを薄いラッパーにし、実処理を再利用可能ワークフローとして切り出す案もあるが、そのラッパー自体が各ブランチのworking treeに存在する必要がある点は変わらないため、上記1・2の運用を代替するものではなく補完に留まる。

## 5. 本ブランチ側の現状

- 本ブランチ（PrivacyCamera）は、mainのRoborazziテスト基盤をまだ取り込んでいない。
- 本ブランチ側で今後どう追従するか（mainをマージするか、当面は独自の軽量静的チェックのみで運用するか）は、本レポートとは別に判断する。
