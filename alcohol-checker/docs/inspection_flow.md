# 非対面アルコール検査フローチャート

> **対象**: 職業ドライバーの乗務前後アルコール検査（非対面点呼対応版）  
> **記録物**: インカメラ動画 ＋ BAC 測定値 ＋ GPS ＋ タイムスタンプ  
> **判定基準**: 呼気中アルコール濃度 0.15 mg/L（道路交通法施行令第 44 条の 3）

## 2. エラー・例外フロー

```
Phase 3 計測中に BLE 切断
─────────────────────────
  BLE 切断イベント
      │
      ▼
  計測キャンセル
  録画停止
      │
      ▼
  「デバイスが切断されました。再接続して再試行してください」
      │
      ▼
  Phase 0 へ戻る


計測中にカメラが遮断された / 顔が外れた
─────────────────────────────────────
  (オプション実装) FaceDetector で顔検出が 2 秒以上外れた場合
      │
      ▼
  警告表示「カメラに顔を向けてください」
  3 秒以上改善なし → 計測中断・再試行を促す


再検査上限 (例: 3 回) 超過
───────────────────────────
  retry_count >= MAX_RETRY (3)
      │
      ▼
  「本日のアルコール検査を完了できませんでした」
  管理者への連絡案内
  以降の操作をロック (任意)
```

## 4. アルゴリズム設計（実装方針）

### 4.1 ステート管理

現行の `MeasureState` を以下に拡張します。

```kotlin
enum class InspectionState {
    IDLE,              // チェック待機
    FACE_GUIDE,        // インカメラ顔確認
    RECORDING_STARTED, // 録画開始 → カウントダウン
    MEASURING,         // BLE 計測中 (5 秒)
    JUDGING,           // 判定処理
    COMPLETE,          // 結果表示 + 備考入力
    SAVED,             // 保存完了
    ERROR              // BLE 切断 / 権限エラー
}
```

### 4.2 録画セッション

```
CheckSession (1回の検査に対応するデータクラス)
├── sessionId: UUID
├── driverId / driverName
├── checkType (乗務前/後)
├── recordingStartAt: Long        ← 録画開始時刻
├── measureStartAt: Long          ← START コマンド送信時刻
├── peakBac: Float                ← 5秒間のピーク値
├── isPassed: Boolean
├── videoFilePath: String?        ← 録画ファイルパス
├── latitude / longitude: Double?
└── retryCount: Int               ← 再検査回数
```

### 4.3 各フェーズのタイミング

```
T=0.0s  FACE_GUIDE 開始 → 録画スタート
T=X.Xs  ユーザーが「準備完了」をタップ (または 5秒後自動)
T=X.Xs  カウントダウン表示開始: 3 → 2 → 1 (各 1 秒)
T=X+3s  COUNT=0 → BLE START 送信 → 計測タイマー開始
T=X+8s  計測完了 → peak_bac 確定 → 判定
T=X+8+αs 録画停止 (判定結果が画面に入るように +2 秒余裕)
```

### 4.4 BLE ↔ 動画の同期戦略

```
┌──────────────────────────────────────────────┐
│ BLE START コマンド送信と同じフレームで        │
│ measureStartAt = SystemClock.elapsedRealtime() を記録
│                                              │
│ 動画ファイルには recordingStartAt も記録      │
│ → 差分から「動画内の何秒時点が計測開始か」    │
│   を後から特定可能                           │
└──────────────────────────────────────────────┘
```

### 4.5 再検査ロジック

```
合格: retryCount にかかわらず保存 → IDLE
不合格:
  retryCount < MAX_RETRY (推奨 3):
    retryCount++
    「再検査しますか？」ダイアログ
    YES → 現セッションは DB 保存 (不合格記録として残す)
          新セッション開始 → FACE_GUIDE へ戻る
    NO  → 管理者連絡画面
  retryCount >= MAX_RETRY:
    「上限に達しました。管理者に連絡してください。」
    それ以上のチェックをブロック (当日 or 手動解除まで)
```

### 4.6 動画ファイル管理

```
保存場所: Context.filesDir/videos/
ファイル名: check_{sessionId}_{timestamp}.mp4
FileProvider スコープ: 既存の file_paths.xml に videos/ を追加
共有: CSV エクスポート時に動画も一緒に ZIP 圧縮して共有可能にする（将来拡張）
```
