# 非対面アルコール検査フローチャート（改訂版）

> **対象**: 職業ドライバーの乗務前後アルコール検査（非対面点呼対応）  
> **記録物**: スーパーインポーズ済み動画 ＋ BAC 測定値 ＋ GPS ＋ タイムスタンプ  
> **判定基準**: 呼気中アルコール濃度 0.15 mg/L（道路交通法施行令第 44 条の 3）

---

## 2. 健康状態 自動評価ロジック

```
入力: 6 項目の回答 (normal / abnormal)

  if 体温 == 発熱
   OR 服薬 == あり(眠気副作用)
      → healthStatus = "乗務不可"
         → 警告ダイアログ → 管理者連絡へ
         → アルコール検査へ進ませない

  else if 頭痛・めまい == あり
       OR 吐き気・腹痛 == あり
       OR 倦怠感 == あり
       OR 睡眠 == 不足
      → healthStatus = "要注意"
         → 警告付きで検査継続 (管理者通知を推奨)

  else
      → healthStatus = "良好"
```

## 3. スーパーインポーズ仕様

### 3.1 映像内レイアウト

```
┌──────────────────────────────────────────────────────────────────┐
│ ┌──────────────────────────────────────────────────────────────┐ │
│ │ 氏名: 田中 太郎   社員ID: D-001                              │ │  ← 上部左 (白文字/黒背景半透明)
│ │ 2026-06-28  07:45:23  JST                                    │ │
│ │ 場所: 東京都港区芝浦 1-1-1  (35.6500, 139.7570)             │ │
│ └──────────────────────────────────────────────────────────────┘ │
│                                                                   │
│                  [ カメラ映像 ]                                   │
│                                                                   │
│ ┌──────────────────────────────────────────────────────────────┐ │
│ │ 健康状態: 良好     天候: 晴れ 気温 28℃                       │ │  ← 下部左
│ │ アルコール検査数値: 0.032 mg/L   ┃  判定: 合格 ✓             │ │  ← 下部右
│ │ 検査種別: 乗務前              ┃  基準値: 0.15 mg/L未満       │ │
│ └──────────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────────┘
```

### 3.2 各項目の情報源

| 表示項目 | 情報源 | 更新タイミング |
|---------|--------|---------------|
| 氏名・社員ID | Driver DB | 計測開始時に固定 |
| 日時 | `System.currentTimeMillis()` | 毎秒更新（録画中） |
| 場所（住所） | GPS + Geocoder逆ジオコーディング | 録画開始時に固定 |
| GPS 座標 | `FusedLocationProviderClient` | 録画開始時に固定 |
| 健康状態 | 問診回答の自動評価 | 問診完了時に固定 |
| 天候・気温 | OpenWeatherMap API or 手動選択 | 録画開始時に固定 |
| アルコール数値 | BLE notify (peak_bac) | 計測中はリアルタイム、完了後に固定 |
| 判定 | peak_bac vs 0.15 | 計測完了後に固定（"計測中" → "合格/不合格"） |

### 3.3 スーパーインポーズ実装方式

**方式 A（推奨・実装優先）: 後処理合成**

```
録画フロー:
  CameraX VideoCapture (インカメラ)
        │
        ▼
  raw_{sessionId}.mp4 (純粋な映像)
        │
        ▼ 計測完了後にバックグラウンド処理
  VideoCompositor.compose()
    ├─ MediaExtractor でフレームごとに取り出す
    ├─ ImageReader → Bitmap に変換
    ├─ Canvas.drawBitmap() でカメラフレームを描画
    ├─ Canvas.drawText/drawRect() でオーバーレイ描画
    │    ├─ 上部バー: 氏名, 日時 (タイムスタンプから再計算)
    │    ├─ 下部バー: 健康状態, 天候, BAC, 判定
    │    └─ タイムコード: 動画秒数 + recordingStartAt
    ├─ MediaCodec で再エンコード
    └─ MediaMuxer で final_{sessionId}.mp4 として書き出し

処理時間目安: 10秒動画 → 5〜15秒 (端末性能による)
```

**方式 B（高品質・将来拡張）: リアルタイム OpenGL 合成**

```
録画フロー:
  Camera2 API
      │ SurfaceTexture (OES テクスチャ)
      ▼
  EGLContext (OpenGL ES 2.0)
      ├─ カメラフレームをテクスチャとして描画
      └─ オーバーレイ Canvas (GLSurfaceView のコールバック内で描画)
            │ MediaRecorder 用 Surface に出力
            ▼
  MediaRecorder → final_{sessionId}.mp4
  (リアルタイム合成のため後処理不要)

品質: 最高  処理遅延: なし  実装複雑度: 高
```

**方式選択指針**:

```
実装フェーズ 1 → 方式 A (後処理合成)
  - CameraX が使えるため既存コードを活用しやすい
  - 計測後に合成するので録画には影響なし
  - 動画確認まで数秒〜十数秒の待機が発生

実装フェーズ 2 → 方式 B (OpenGL リアルタイム)
  - 合成遅延なし、管理者がリアルタイムモニタリング可能
  - カスタム CameraView が必要
```

## 4. 更新版ステートマシン

```
IDLE
  │ onStartCheck()
  ▼
FACE_GUIDE          ← インカメラ ON / 録画準備
  │ onReadyTapped() or 5秒自動
  ▼
RECORDING_STARTED   ← ★録画開始 / GPS・天候取得
  │ 即座に
  ▼
HEALTH_QUESTIONNAIRE← 問診票表示（録画中）
  │ onHealthSubmitted()
  ├─ healthStatus == "乗務不可" ──▶ HEALTH_BLOCKED
  ▼
COUNTDOWN           ← 3・2・1 ビープ
  │ count == 0 → BLE START 送信
  ▼
MEASURING           ← 5秒間計測・BAC リアルタイム表示
  │ 5秒経過
  ▼
JUDGING             ← peak_bac 確定・判定
  │ 判定完了 (録画は +2秒維持)
  ▼
RECORDING_STOPPED   ← ★録画停止
  │ バックグラウンドで後処理合成開始
  ▼
COMPOSITING         ← スーパーインポーズ合成中 (ローディング表示)
  │ 合成完了
  ▼
COMPLETE            ← 結果表示 / 備考入力
  │ onSave()
  ▼
SAVED               ← DB保存完了

エラー分岐:
  MEASURING 中 BLE 切断 ──▶ ERROR (計測キャンセル・録画停止)
  HEALTH_BLOCKED         ──▶ 管理者連絡画面
```

## 6. 天候取得方針

```
優先順位:

1. OpenWeatherMap API (無料枠: 1,000 call/日)
   → 現在地の天候・気温を自動取得
   → GPS 座標を API に送信
   → APIキーを BuildConfig 経由で管理 (git には含めない)

2. 手動選択フォールバック
   → API 失敗時 / オフライン時
   → プルダウン: 晴れ / 曇り / 雨 / 雪 / 霧
   → 任意入力の気温欄

API エンドポイント例:
  GET https://api.openweathermap.org/data/2.5/weather
      ?lat={lat}&lon={lon}&appid={key}&units=metric&lang=ja
```
